## 🍳 1. What is a Stored Procedure?
Think of a stored procedure as a saved recipe!

* Normal SQL: Cooking every time from scratch.
* Procedure: Save the recipe once, and just call it whenever you need it!

## 💡 Real-Life Use Cases

* Daily Reports: Automate repetitive queries.
* Complexity: Wrap massive 50-line queries into a single command.
* Consistency: Ensure multiple developers use the exact same logic.

## 📝 Basic Syntax

DELIMITER $$                    -- 1. Temporarily change the delimiter!
CREATE PROCEDURE procedure_name()BEGIN
    -- Your SQL logic goes here
    SELECT * FROM employees;END $$

DELIMITER ;                     -- 2. Restore the original delimiter!
-- 🚀 Call it anytime!CALL procedure_name();

------------------------------
## 🧠 2. Why do we need a DELIMITER?
Normally, MySQL uses a semicolon ; to end a statement. However, inside a procedure, ; is used to end each line of code. Without changing the delimiter, MySQL gets confused about where the procedure actually ends!

* Solution: Temporarily change ; to $$ so MySQL knows to read the whole block as one unit.

DELIMITER $$          -- Now $$ ends the entire statement!
CREATE PROCEDURE ...BEGIN
    SELECT * FROM employees;   -- ; is now safe to use inside!
    SELECT * FROM departments; -- ; is now safe to use inside!END $$                         -- $$ signals the end of the procedure!

DELIMITER ;           -- Restore back to the normal semicolon!

------------------------------
## 📥 3. Types of Parameters
MySQL procedures support three types of parameters to pass data in and out.
## ➡️ IN (Input Only)
Passes a value TO the procedure.

CREATE PROCEDURE get_emp_by_dept(IN dept_id INT)BEGIN
    SELECT * FROM employees
    WHERE dept_id = dept_id;END $$
-- Execution:CALL get_emp_by_dept(3);  -- Passes dept_id = 3

## ⬅️ OUT (Output Only)
Sends a value BACK to the caller.

CREATE PROCEDURE get_emp_count(
    IN dept_id INT,
    OUT emp_count INT
)BEGIN
    SELECT COUNT(*) INTO emp_count
    FROM employees
    WHERE dept_id = dept_id;END $$
-- Execution:CALL get_emp_count(3, @count);SELECT @count;  -- Displays the output

## 🔄 INOUT (Both Input and Output)
Passes a value in, modifies it, and returns it.

CREATE PROCEDURE double_salary(INOUT salary INT)BEGIN
    SET salary = salary * 2;END $$
-- Execution:SET @sal = 50000;CALL double_salary(@sal);SELECT @sal;  -- Returns 100000

------------------------------
## 📌 4. Variables Inside Procedures
You can declare local variables using the DECLARE keyword. They must be placed at the very beginning of the BEGIN...END block.

CREATE PROCEDURE salary_stats()BEGIN
    DECLARE avg_sal DECIMAL(10,2);   -- Declare first!
    DECLARE max_sal DECIMAL(10,2);

    SELECT AVG(salary) INTO avg_sal  -- Store calculated value!
    FROM employees;

    SELECT MAX(salary) INTO max_sal
    FROM employees;

    SELECT avg_sal, max_sal;         -- Display results!END $$

------------------------------
## 🔀 5. IF - ELSE Conditional Logic
Stored procedures allow you to implement procedural control flow directly in your database.

CREATE PROCEDURE check_salary(IN emp_id INT)BEGIN
    DECLARE emp_sal DECIMAL(10,2);

    SELECT salary INTO emp_sal
    FROM employees
    WHERE emp_id = emp_id;

    IF emp_sal > 70000 THEN
        SELECT 'High Salary' AS category;
    ELSEIF emp_sal > 40000 THEN
        SELECT 'Medium Salary' AS category;
    ELSE
        SELECT 'Low Salary' AS category;
    END IF;END $$

## ⚖️ 6. Procedures vs. Functions
Understanding the core differences between the two is crucial for database design.

| Feature | 🛠️ PROCEDURE | 🧮 FUNCTION |
|---|---|---|
| Invocation | CALL proc() | SELECT func() |
| Usage in SELECT | Cannot be used in SELECT | Can be used in SELECT queries ✅ |
| Return Value | Returns nothing (uses OUT) | MUST return a single value |
| Statement Type | Handles multiple operations | Best for single scalar returns |
| DML Operations | Allowed (INSERT, UPDATE) | Discouraged / Usually read-only |

## 📝 Function Syntax

DELIMITER $$
CREATE FUNCTION function_name(param datatype)RETURNS datatype                 -- Must declare return type!DETERMINISTIC                    -- Same input always = same outputBEGIN
    DECLARE result datatype;
    -- Logic here
    RETURN result;               -- Must return a value!END $$

DELIMITER ;
-- 🚀 Use it like a native SQL function!SELECT function_name(column) FROM table;

## 📆 7. Real-World Function Example
A function to calculate an employee's experience in years based on their joining date.

DELIMITER $$
CREATE FUNCTION get_experience(join_date DATE)RETURNS DECIMAL(5,2)DETERMINISTICBEGIN
    DECLARE exp DECIMAL(5,2);
    SET exp = DATEDIFF(CURDATE(), join_date)/365;
    RETURN ROUND(exp, 2);END $$

DELIMITER ;
-- 🚀 Use it directly inside a query!SELECT name, get_experience(join_date) AS experienceFROM employees;

## 💬 9. Crack the Interview!
Q: What is the difference between a procedure and a function?

A function must return a value and can be used directly inside standard SELECT statements. A procedure is invoked using the CALL statement and can yield multiple values through OUT parameters.

Q: What is a DELIMITER and why is it needed?

It temporarily changes the statement terminator inside MySQL. This prevents the compiler from mistaking the internal semicolons ; of the procedure block for the end of the entire procedure creation statement.

Q: What does DETERMINISTIC mean?

It guarantees that the function will always produce the exact same output when given the exact same input parameters. The opposite is NOT DETERMINISTIC (e.g., functions dealing with current timestamps or random numbers).

Q: Can a function call a procedure?

No, a function cannot call a procedure because functions cannot perform actions that modify the database state. However, a procedure is perfectly capable of calling a function!

Q: Where are procedures stored?

They are stored directly within the database server catalog itself, not within the backend application code!

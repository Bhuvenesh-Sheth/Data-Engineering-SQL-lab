1️⃣ WHAT IS A STORED PROCEDURE?
Think of it like a SAVED RECIPE! 🍳

Normal SQL = cooking every time from scratch
Procedure  = save the recipe, just call it!

Real life use:
→ Run same report daily? Save as procedure!
→ Complex 50-line query? Save as procedure!
→ Multiple people need same logic? Procedure!
sql-- BASIC SYNTAX
DELIMITER $$                    -- change delimiter!

CREATE PROCEDURE procedure_name()
BEGIN
    -- your SQL logic here
    SELECT * FROM employees;
END $$

DELIMITER ;                     -- restore delimiter!

-- CALL it anytime!
CALL procedure_name();

2️⃣ WHY DELIMITER?
sql-- 🧠 Normally SQL uses ; to end statements
-- But inside procedure, ; ends each line
-- MySQL gets confused which ; ends procedure!

-- Solution: Change ; to $$ temporarily!

DELIMITER $$          -- now $$ ends statements!

CREATE PROCEDURE ...
BEGIN
    SELECT * FROM employees;   -- ; safe inside now!
    SELECT * FROM departments; -- ; safe inside now!
END $$                         -- $$ ends procedure!

DELIMITER ;           -- restore back to normal!

3️⃣ TYPES OF PARAMETERS
sql-- IN  → input you pass TO procedure
-- OUT → output procedure sends BACK to you
-- INOUT → both input and output!

-- IN parameter example:
CREATE PROCEDURE get_emp_by_dept(IN dept_id INT)
BEGIN
    SELECT * FROM employees
    WHERE dept_id = dept_id;
END $$
-- Call it:
CALL get_emp_by_dept(3);  -- pass dept_id = 3!

-- OUT parameter example:
CREATE PROCEDURE get_emp_count(
    IN dept_id INT,
    OUT emp_count INT
)
BEGIN
    SELECT COUNT(*) INTO emp_count
    FROM employees
    WHERE dept_id = dept_id;
END $$
-- Call it:
CALL get_emp_count(3, @count);
SELECT @count;  -- see the output!

-- INOUT parameter example:
CREATE PROCEDURE double_salary(INOUT salary INT)
BEGIN
    SET salary = salary * 2;
END $$
-- Call it:
SET @sal = 50000;
CALL double_salary(@sal);
SELECT @sal;  -- returns 100000!

4️⃣ VARIABLES INSIDE PROCEDURES
sql-- Declare variables with DECLARE
-- Must be declared FIRST inside BEGIN!

CREATE PROCEDURE salary_stats()
BEGIN
    DECLARE avg_sal DECIMAL(10,2);   -- declare first!
    DECLARE max_sal DECIMAL(10,2);

    SELECT AVG(salary) INTO avg_sal  -- store value!
    FROM employees;

    SELECT MAX(salary) INTO max_sal
    FROM employees;

    SELECT avg_sal, max_sal;         -- show results!
END $$

5️⃣ IF ELSE INSIDE PROCEDURES
sqlCREATE PROCEDURE check_salary(IN emp_id INT)
BEGIN
    DECLARE emp_sal DECIMAL(10,2);

    SELECT salary INTO emp_sal
    FROM employees
    WHERE emp_id = emp_id;

    IF emp_sal > 70000 THEN
        SELECT 'High Salary' AS category;
    ELSEIF emp_sal > 40000 THEN
        SELECT 'Medium Salary' AS category;
    ELSE
        SELECT 'Low Salary' AS category;
    END IF;
END $$

6️⃣ WHAT IS A FUNCTION?
Procedure vs Function — THE KEY DIFFERENCE!

╔══════════════════════════════════════════════╗
║  PROCEDURE              FUNCTION             ║
╠══════════════════════════════════════════════╣
║  CALL proc()            SELECT func()        ║
║  Can't use in SELECT    Used in SELECT! ✅   ║
║  Returns nothing/OUT    MUST return value!   ║
║  Multiple statements    Single return value  ║
║  DML allowed (INSERT)   Usually no DML       ║
╚══════════════════════════════════════════════╝
sql-- FUNCTION SYNTAX
DELIMITER $$

CREATE FUNCTION function_name(param datatype)
RETURNS datatype                 -- must declare!
DETERMINISTIC                    -- same input = same output!
BEGIN
    DECLARE result datatype;
    -- logic here
    RETURN result;               -- must return!
END $$

DELIMITER ;

-- USE it like any SQL function!
SELECT function_name(column) FROM table;

7️⃣ REAL FUNCTION EXAMPLE
sql-- Function to calculate experience in years
DELIMITER $$

CREATE FUNCTION get_experience(join_date DATE)
RETURNS DECIMAL(5,2)
DETERMINISTIC
BEGIN
    DECLARE exp DECIMAL(5,2);
    SET exp = DATEDIFF(CURDATE(), join_date)/365;
    RETURN ROUND(exp, 2);
END $$

DELIMITER ;

-- Use it directly in SELECT!
SELECT name, get_experience(join_date) AS experience
FROM employees;

8️⃣ DROP & SHOW (Maintenance!)
sql-- See all procedures
SHOW PROCEDURE STATUS WHERE db = 'your_db';

-- See all functions
SHOW FUNCTION STATUS WHERE db = 'your_db';

-- Delete procedure
DROP PROCEDURE IF EXISTS procedure_name;

-- Delete function
DROP FUNCTION IF EXISTS function_name;

-- See procedure code
SHOW CREATE PROCEDURE procedure_name;

9️⃣ INTERVIEW QUESTIONS ON THIS TOPIC!
Q: Difference between procedure and function?
A: Function returns value, used in SELECT.
   Procedure called with CALL, can have OUT params.

Q: What is DELIMITER and why needed?
A: Changes statement terminator inside procedure
   so ; inside BEGIN doesn't confuse MySQL.

Q: What is DETERMINISTIC?
A: Same input always gives same output.
   Opposite is NOT DETERMINISTIC (random/time based)

Q: Can function call procedure?
A: No! But procedure can call function!

Q: Where are procedures stored?
A: In database itself — not in application code!
